# 任务决策提示词

## 任务
根据用户输入，判断需要执行什么类型的诊断任务。

## 输入
用户问题：{user_query}
是否上传了图像：{has_image}

## 任务类型定义

**greeting**：用户只是打招呼或做身份确认，如“你好”“您好”“hi”“hello”“在吗”“你是谁”

**image_only**：有图像，但没有具体症状描述；或者文字只是泛泛询问，如“这是什么”“帮我看看”“请问”这类，不算症状

**symptom_only**：只有文字描述症状，没有图像，例如“背上长红斑很痒”“手上脱皮怎么办”

**symptom_and_image**：同时有图像和具体症状描述，例如“背上长红斑很痒”并上传了图片

**follow_up**：没有新图像，且用户是在延续上一轮诊断、补充说明或追问，例如“还是疼”“那怎么办”“继续看看”“还有别的症状吗”

## 判断规则
1. 如果只是打招呼/问身份 → greeting
2. 如果有图像，且文字是泛泛询问，没有具体症状 → image_only
3. 如果有图像，且文字包含具体症状 → symptom_and_image
4. 如果没有图像，且是具体症状描述 → symptom_only
5. 如果没有新图像，且明显是在追问或补充上一轮诊断 → follow_up

## 关键点
- “这是什么症状”“帮我看看”“请问” → 这是询问，不算症状描述 → image_only
- “背上长红斑很痒” → 这是具体症状 → symptom_and_image
- “你好”“你是谁”“在吗” → greeting
- “还是疼”“那怎么办”“继续看看” → follow_up
- 如果当前没有新图像，不要因为历史里有图片就判成 image_only 或 symptom_and_image；历史图片只用于 follow_up 背景判断

## 输出
只输出一个词：
- greeting
- image_only
- symptom_only
- symptom_and_image
- follow_up
